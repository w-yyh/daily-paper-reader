## 摘要
领域泛化旨在从多个源域学习并泛化到未见的目标域，然而它往往忽略了源域与目标域标签不匹配的现实情况。为此，开放集领域泛化被提出，用于在未见域中识别未见类。一种简单的方法是训练一对多分类器来分离每个类别并将异常值检测为未知。但少量正样本与大量负样本之间的不平衡会使决策边界偏向正样本，导致模型过度拒绝分布外数据，甚至包括未见域中的已知类别。本文提出了一种新颖的元学习策略，称为联合域-类匹配的二元元学习（MEDIC），它同时考虑域间和类间任务划分的隐式梯度匹配，以寻找在域和类上都平衡的最优边界。实验结果表明，MEDIC不仅在开放集场景中优于现有方法，而且保持了有竞争力的闭集泛化能力。

## Abstract
Domain generalization learns from multiple source domains to generalize to unseen target domains. However, it often neglects the realistic case of label mismatch between source and target. Open set domain generalization is then proposed to recognize unseen classes in unseen domains. A simple approach trains one-vs-all classifiers to separate each class and detect outliers as unknown. Yet, the imbalance between few positive samples and many negative samples skews the decision boundary towards the positive ones, leading the model to over-reject out-of-distribution data, even from known classes in unseen domains. In this paper, we propose a novel meta-learning stategy called dualistic MEta-learning with joint DomaIn-Class matching (MEDIC), which considers implicit gradient matching towards inter-domain and inter-class task splits simultaneously to find optimal boundaries balanced for both domains and classes. Experimental results show that MEDIC not only outperforms prior methods in open set scenarios, but also maintains competitive close set generalization ability.

*   **研究背景与动机**：领域泛化旨在从多个源域学习并泛化到未见的目标域。然而，现实部署中，目标域不仅数据分布未知，还可能包含训练时从未见过的**新类别**。这种同时处理分布漂移（Domain Shift）和类别漂移（Category Shift）的更实际设定被称为**开放集领域泛化**。
*   **核心问题**：现有方法常采用一对多分类器识别未知类，但训练时正样本（目标类别）稀少、负样本（所有其他类别）丰富，导致**决策边界严重偏向于正样本**。这会将许多来自新域的**已知类样本过度拒绝为“未知”**，严重损害模型在新域上的泛化性能与安全性。
*   **整体含义**：本文提出核心观点：要解决上述边界偏斜问题，需要在模型优化过程中，不仅平衡不同**域**之间的更新方向，也要平衡不同**类别**之间的更新方向，从而学习到域不变且类平衡的决策边界。

### 2. 方法论：核心思想、关键技术细节

论文提出了一种名为**MEDIC**的元学习策略。其核心思想是利用**隐式梯度匹配**，在域和类两个维度上同时寻找一个“双赢”的优化方向。

*   **核心思想——双元梯度匹配**：
    *   常规域泛化元学习方法（如MLDG、Fish）仅在域级别进行任务划分与梯度匹配，以学习域不变特征，但忽略了类别关系。
    *   MEDIC在此基础上进一步在**类别级别**划分任务，并重构元训练和元测试任务对。
    *   **目标函数设计**：通过巧妙重组，让每一次模型更新的梯度方向，不仅与来自不同域的任务梯度对齐（域匹配），也与包含不同类别的任务梯度对齐（类匹配）。
*   **关键技术细节与公式框架**：
    *   **任务划分**：将源域数据同时按域和类切分为多个子任务，构成任务池。
    *   **元学习流程**：采用类似Reptile的一阶元学习范式，分内循环和外循环。
        *   **内循环**：分为多个步骤（Steps）。每个步骤随机采样多个任务（Tasks）进行连续梯度更新。**MEDIC++**作为通用框架，通过调控步数与每步任务数的比例，在计算效率和梯度匹配全面性之间权衡。
        *   **梯度匹配机理**：通过泰勒展开，证明了该元目标等价于最大化不同任务梯度间的点积，即最小化它们之间的夹角。
    *   **自适应任务采样**：为了更高效地解决易混淆类别之间的边界偏斜，引入基于类别转移概率的自适应采样策略。模型记录分类混淆矩阵，让高频互误的类别对更频繁地出现在不同任务中，以强化它们之间的梯度匹配。
    *   **损失函数**：结合传统的交叉熵损失和专为开放集设计的**一对多损失**。一对多损失不仅最大化目标类别的正输出概率，还最小化最难负类别的输出概率，从而训练出清晰的每个已知类别的决策边界。
    *   **推断过程**：测试时，先用闭集分类器预测已知类的概率分布，再通过其最大概率类对应的一对多分类器的正输出通道得分与预设阈值比较，低于阈值则判为未知类。
