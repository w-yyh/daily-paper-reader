## 摘要
无监督持续学习（UCL）旨在使神经网络能够在没有标签或无法访问过去数据的情况下学习顺序任务。此设置中的一个主要挑战是灾难性遗忘，即模型在学习新任务时忘记先前学习的任务。由于缺乏标签来指导学习和记忆保留，这一挑战在UCL中被放大。现有的缓解策略，如知识蒸馏和重放缓冲区，常常引发内存和隐私方面的担忧。此外，当前的UCL方法在很大程度上忽略了聚类特定的目标。为填补这一空白，我们引入了无监督持续聚类（UCC），并提出了用于持续聚类的前向-后向知识蒸馏（FBCC）。FBCC采用一个具有聚类投影器的持续教师网络和轻量级任务特定学生网络。通过双阶段的前向-后向蒸馏过程，教师在不存储过去数据的情况下学习新簇，同时保留先前发现的簇结构。FBCC代表了UCC的一种开创性方法，展示了在顺序任务上改进的聚类性能。在四个基准数据集上的实验表明，FBCC在聚类准确性上持续优于现有的持续学习基线，同时显著减少了灾难性遗忘。

## Abstract
Unsupervised Continual Learning (UCL) aims to enable neural networks to learn sequential tasks without labels or access to past data. A major challenge in this setting is Catastrophic Forgetting, where models forget previously learned tasks upon learning new ones. This challenge is amplified in UCL due to the absence of labels to guide learning and memory retention. Existing mitigation strategies, such as knowledge distillation and replay buffers, often raise memory and privacy concerns. Moreover, current UCL methods largely overlook clustering-specific objectives. To fill this gap, we introduce Unsupervised Continual Clustering (UCC) and propose Forward-Backward Knowledge Distillation for Continual Clustering (FBCC). FBCC employs a continual teacher network with a clustering projector and lightweight task-specific students. Through a dual-phase forward-backward distillation process, the teacher learns new clusters while preserving previously discovered cluster structure without storing past data. FBCC represents a pioneering approach to UCC, demonstrating improved clustering performance across sequential tasks. Experiments on four benchmark datasets demonstrate that FBCC consistently outperforms existing continual learning baselines in clustering accuracy while significantly reducing catastrophic forgetting.

### 1. 研究背景与核心问题
- **研究动机**：无监督持续学习（UCL）要求模型在无标签、无历史数据回放的情况下，顺序学习多个任务。其中的核心挑战是**灾难性遗忘**，即模型在学习新任务时急剧丧失对旧任务的辨识能力。
- **研究空白**：现有UCL方法主要聚焦于**表征学习**，而聚类仅作为下游评估手段，并未将聚类本身作为持续学习的核心目标。同时，常用的知识蒸馏和重放缓冲区方法存在内存负担重、隐私泄露等缺陷，且缺乏专门针对聚类的持续学习框架。
- **问题定义**：本文首次提出**无监督持续聚类（UCC）**设定，要求模型在顺序任务流中，不仅学习表征，还要同时发现并维持每个任务的簇结构，且不能访问旧任务数据或标签。

### 2. 方法论
论文提出**前向-后向知识蒸馏持续聚类（FBCC）** 框架，其核心组件与流程如下：
- **整体架构**：
  - **教师网络**：参数较大的主网络（如ResNet-18），附带一个**聚类投影器**，用于将表征映射到聚类概率空间。
  - **学生网络**：轻量级任务专用网络（如SqueezeNet 1.1），参数量远小于教师，专门模仿教师对某个历史任务的行为。
- **前向蒸馏（Forward Distillation）**：训练教师学习当前任务，同时保留旧知识。
  - **对比学习损失**（`L_con`）：利用样本的不同增强视图，强制同一实例的表征相似，并借助旧任务**原型集（prototypes）** 将当前任务表征与旧任务表征推开。
  - **知识蒸馏损失**（`L_dis`）：将当前任务样本的教师表征，通过可学习的**预测器**对齐到冻结的旧学生网络输出的表征上，从而约束教师不遗忘旧任务。
  - **聚类损失**（`L_clu`）：在聚类投影器的输出上施加簇级对比损失，使同类簇概率分布相近，异类簇远离，并引入簇分配熵最大化以避免平凡解。
  - **原型更新**：每任务结束后，计算可靠样本的簇中心作为新原型，加入原型集供后续任务使用。
- **后向蒸馏（Backward Distillation）**：训练当前任务的学生网络，以复现教师对当前任务的行为。
  - **模仿损失**（`L_stu`）：包含两部分：（1）学生与教师中间表征的**MSE对齐**；（2）学生与教师输出表征之间的**对比损失**，促使学生继承教师的样本间结构关系。
- **内存管理**：只保留最近`M`个学生模型（`M = ⌈N/2⌉`），极大减少内存开销。
- **最终簇分配**：完成所有任务后，利用存储的任务特定聚类投影器最后一层，将所有样本经过教师和共享的投影器第一层后，分配到得分最高的簇。

### 7. 优点
- **首创性**：首次明确定义无监督持续聚类（UCC），并设计从学习到评估的完整流程。
- **创新蒸馏机制**：双向知识蒸馏，教师通过学生记忆巩固旧知识，学生通过模仿教师掌握当前任务，形成闭环，无需重放旧数据。
- **内存与隐私友好**：无重放缓冲区，仅保留轻量学生和簇投影头，避免存储原始样本，适用于隐私受限场景。
- **实验条件严格**：对比方法涵盖UCL和SCL，使用统一的评估协议，并提供多组消融和鲁棒性分析，实证严谨。
- **高效与高性能兼得**：在较低模型大小和合理训练时间下，超越更大规模的重放或双模型方法。

### 8. 不足与局限
- **簇数需预知**：设定中假设每个任务的簇数量已知，而实际流式数据中簇数可能未知或动态变化，限制了方法的应用范围。
- **任务边界假设**：方法依赖明确的任务边界（task incremental），在无边界的数据流（task-free）场景下，如何触发学生训练和原型更新尚未讨论。
- **类别与簇的对应**：评估时将簇数设为真实类别数，但无监督聚类产生的簇可能与语义类别不匹配，评价指标无法完全反映实际聚类纯度。
- **超参数敏感性**：学生保留数量`M`需依赖总任务数`N`来设定，对于完全未知任务总数的情况不够鲁棒。
